## What was done

### A. chrome-ext 0.1.4 ship-blocker fix — nested-selector double-match (committed before this session resumed, but the verification was here)

`querySelectorAll(primary + ", " + fallback)` matched BOTH tiers when Anthropic's response DOM was nested (`.font-claude-response` containing `.standard-markdown`). For-loop iterated twice → different `i` → different dedupe keys → two `browser.response` events ~2 ms apart with identical text. Fix: use the pre-existing `resolveAll(tier)` helper which only falls back to the secondary tier if the primary returned zero matches. Applied to claude.ts capturePromptsFromDOM / captureResponses / captureToolCalls and the parallel two callsites in chatgpt.ts. Five touch-points total. Commit `ca60ed9`.

NOT changed: the legitimate `turn?.querySelector(primary) || turn?.querySelector(fallback)` pattern — that's the correct cascade in single-element scope.

Live retest blocked by Anthropic outage. Project-tag verification done via a KONIVE.com probe instead — confirmed pipeline + hook + cwd-tagging all work end-to-end (event landed at `2026-06-23T14:59:24` with `cwd=/Users/yan/Projects/KONIVE.com`).
