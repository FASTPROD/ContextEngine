### Commit `7bf34b6` — `feat(policy): declarative policy.json contract`
P1 #4 part 1 — the contract that hooks/CC/CI all consume.

- **`src/policy.ts`** (LOCK `[POLICY-CONTRACT]`) — v1 schema with
  four sections:
  - `secret_patterns` — id-tagged regex rules, severity `block`|`warn`,
    optional `paths` glob scoping
  - `doc_coverage` — source-subtree → doc-section mappings (replaces
    the 4h wall-clock staleness gate with diff-aware coverage)
  - `deploy_verify_hosts` — production hosts requiring a verification
    probe within N seconds of git push
  - `bypass_tokens` — documented escape hatches with reason + TTL
    (alternative to undocumented `--no-verify`)
- Zod-based `validatePolicy()` returns a typed `Policy` or a structured
  list of field-level errors. `parsePolicy()` adds the JSON-parse layer.
- `loadRepoPolicy(repoRoot)` reads `.contextengine/policy.json`,
  returns null when absent so hooks fall back to defaults.
- **CLI** — `contextengine policy validate <file>` (CI-friendly, 0/1)
  and `contextengine policy show` (loads + pretty-prints the active
  repo policy).
- **CE dogfoods its own policy** — `.contextengine/policy.json` ships
  with 3 secret patterns (jwt_in_session_doc maps to the 2026-04-09
  Apec incident shape; anthropic; openai), 4 doc-coverage rules, 1
  deploy_verify_host (api.compr.ch), 1 bypass_token (emergency_hotfix
  with 30-char reason minimum).
- **18 policy tests** in `tests/policy.test.ts`.

### Commit `18711a0` — `feat(hooks): policy-driven secret-scan + doc-coverage checkers`
P1 #4 part 2 — the TypeScript enforcement path.

- **`src/hooks.ts`** (LOCK `[HOOK-CHECKERS]`):
  - `getStagedFiles(repoRoot)` — parses `git diff --cached --unified=0`
    into structured `{path, addedLines:[{lineNumber, content}]}`
    records. Hunk-header parsing tracks the actual new-file line
    numbers for reporting.
  - `runSecretScan(policy, files)` — applies `policy.secret_patterns`
    with `paths` glob scoping. **Redaction contract**: returns
    `SecretViolation[]` carrying `pattern_id + severity + file + line`
    ONLY. The matched value is NEVER serialized. Verified by a test
    that greps the serialized output for the matched substring.
  - `runDocCoverage(policy, files, repoRoot)` — diff-aware coverage.
    Per rule: did the commit touch a matching source path AND is the
    required doc either staged or present?
  - `hashDocSection(file, anchor)` — stable SHA-256 of the markdown
    section under a header. Foundation for the v2 anchor-hash check
    that will detect actual section content change.
  - `globToRegExp` + `matchesAnyGlob` — tiny in-house glob matcher
    (`**`, `*`, `?`, regex-meta escape). No new dep.
- **CLI** — `contextengine hook secret-scan` and `hook doc-coverage`.
  Exit 0 clean / 1 on blocking violation. `CE_JSON=1` switches to
  one-line JSON for CI logs.
- Every blocking violation appends a `hook.block` record to the
  tamper-evident audit log.
- **25 hook tests** in `tests/hooks.test.ts` — glob matcher exact +
  `**` + `?` + regex-meta + anchoring; `runSecretScan` global +
  scoped + line numbers + the redaction-contract test;
  `runDocCoverage` for all four outcomes; `hashDocSection` mutation
  detection; formatters + JSON parseability; live `getStagedFiles`
  against an ephemeral git repo via `mkdtempSync`.
- **Smoke tested end-to-end** in `/tmp/ce-hook-policy-test` against
  CE's own policy.

### Commit `1075978` — `feat(hooks): pre-commit migration`
P1 #4 part 3 — the production hook now calls the TypeScript checkers.

- **`hooks/pre-commit`** rewritten:
  - `find_ce_cli()` helper locates the CLI via
    `node_modules/.bin/contextengine` → global `command -v
    contextengine`. **No `npx` fallback** — cold npx is 2–3 s, too
    expensive for the hot path.
  - **Layered secret scan**: gitleaks (if installed) → CE
    policy-driven (if CLI + policy.json) → CE inline 17 patterns
    (always). First-to-block wins.
  - **Doc coverage**: when policy is present, run only `contextengine
    hook doc-coverage` and exit. When no policy, run the legacy 4 h
    wall-clock check unchanged so existing installations keep working.
  - Legacy error banner gains a one-line migration nudge:
    `💡 Author .contextengine/policy.json for diff-aware coverage`.
- **Dogfooded**: `cp hooks/pre-commit .git/hooks/pre-commit` was done
  before the commit; the commit's own pre-commit output shows
  `✅ No policy secret patterns matched` and `✅ All doc-coverage
  rules satisfied` — the new hook validated itself on its own commit.
- **Three smoke scenarios** in ephemeral `/tmp` repos:
  1. No policy + missing CE docs → legacy 4 h gate blocks. ✓
  2. Policy + JWT-shape token in `docs/sessions/SESSION_99.md` →
     policy-driven Layer 2 blocks with `[block]
