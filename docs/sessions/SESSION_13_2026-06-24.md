### F. Shared-learnings strategy answered (no code yet)

User: "by design, nevertheless, once anonymized, all the clean learnings will be available to anyone who paid the pro service? can this be secured? we need a server based PRO version I assume?"

Answer: Hybrid recommendation:
- **Tier A (free, viral):** top ~100 "general-developer-pain" learnings → MIT-licensed, free on GitHub. Loss-leader for adoption.
- **Tier B (PRO-gated):** proprietary 800+ corpus → fetched daily from `api.compr.ch` using existing Ed25519 license heartbeat. Cancel subscription → cache goes stale.

Why this works:
- Reuses the existing Ed25519 activation infrastructure (`src/activation.ts:344`) — zero new auth code.
- Each PRO user's library is timestamped; canceling subscription stops freshness, not access (defensible).
- PRO members also CONTRIBUTE BACK (opt-in publish, anonymized, human-reviewed) — the corpus grows under PRO members' care, network effect compounds with subscription longevity.

Honest reality check disclosed: no software scheme prevents a paying subscriber from re-publishing their cache. Defense is continuous freshness + TOS + network effects.

Estimate: ~2 days of work for the full hybrid (server endpoint + redaction CLI + sync client). Sprint-3 candidate.
